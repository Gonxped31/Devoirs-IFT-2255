Tâche 3 : Cas d'utilisation

Cas 1

Cas d'utilisation : Inscription d'un client en tant qu'utilisateur
But : Un client désire créer un compte en tant que fournisseur de Robotix.
Préconditions : Posséder au moins un robot. Tous les robots possédés doivent avoir des numéros de série valides. Le client doit produire au moins un robot.
Acteurs : Client (principal).
Scénario principal
1. Étape 1 : Le client se connecte à Robotix.
2. Étape 2 : Il accède au menu principal.
3. Étape 3 : Il choisit l’option de création de compte utilisateur.
4. Étape 4 : Il complète le champ “Profile” en entrant son nom, son prénom, son courriel, son numéro de téléphone et entre également le nom de sa compagnie.
5. Étape 5 : Le système valide les informations du client et affiche un message de confirmation d’inscription.
6. Étape 6 : Le système affiche le menu d’utilisateur avec les différentes options d’utilisation de robots possibles.
Scénarios alternatifs
5a. Le système rejette certaines informations mal entrées.
 5a.1. Le logiciel affiche un message d’erreur et demande au client d’entrée les données de nouveau.
 5a.2. Le scénario reprend à l’étape 2.
Postconditions : Le nouvel utilisateur est enregistré dans la base de données et a maintenant accès aux différentes options d’utilisateur possible
que voici :

  • Enregistrer un robot,
  • Créer des tâches,
  • Allouer des tâches à un ou plusieurs robots,
  • Visualiser l'efficacité énergétique et l'utilisation de ressources des robots,
  • Afficher les métriques d’une flotte de robots,
  • Participer à divers activités (Jeux, éducation, création),
  • Trouver un fournisseur de composantes pour les robots.

Cas 2

Cas d'utilisation : Inscription d’un client en tant que fournisseur.
But : Un client désire créer un compte en tant que fournisseur de Robotix.
Préconditions : Posséder au moins un robot.
 Tous les robots possédés doivent avoir des numéros de série valides.
 Le client doit produire au moins un robot.
Acteurs : Client (principal).
Scénario principal
1. Étape 1 : Le client se connecte à l’outil.
2. Étape 2 : Il accède au menu principal.
3. Étape 3 : Il choisi l’option de création de compte fournisseur.
4. Étape 4 : Il complète le champ “Profile” en entrant le nom, le prénom, le courriel ainsi que le numéro de téléphone.
5. Étape 5 : Il entre ensuite le type de robot fabriqué, le type de composantes fabriqués et la capacité de fabrication.
6. Étape 6 : Le système valide les informations du client
 6.1 Il affiche un message de confirmation d’inscription si les informations entrées sont valides.
7. Étape 7 : Le système affiche le menu principal de nouveau.
Scénarios alternatifs
6a. Le client ne possède aucun robot (mais il entre quand même 0 comme information).
 6a.1. Le logiciel informe le client du problème.
 6a.2. Le scénario se termine.
6b. Le numéro de série entré n’est pas valide.
   6b.1. Le logiciel informe le client du problème et demande au client d’entrer de nouvelles données.
   6b.2. Le scénario reprend à l’étape 4.
6c. Le client ne produit aucun robot.
 6c.1. Le logiciel informe le client du problème.
 6c.2. Le scénario se termine.
Postconditions : Le nouveau fournisseur sera enregistré dans la base de données.
Il sera également mis dans la même liste de fournisseur que les autres fournisseurs ayant les mêmes caractéristiques que lui.
Ce dernier sera donc présent dans la liste des fournisseurs affiché lorsque l’utilisateur effectue sa recherche de fournisseur.

Cas 3

Cas d'utilisation : Enregistrement d’un robot.
But : Un utilisateur souhaite enregistrer un nouveau robot.
Préconditions : Posséder un numéro de série (provenant d’un fournisseur) et le type du robot à enregistrer.
Acteurs : Utilisateur (principal).
Scénario principal
1. Étape 1 : L’utilisateur choisi l’option d’enregistrement de robot parmi les options disponibles.
2. Étape 2 : Il entre le numéro de série du robot à enregistrer ainsi que le type de ce robot.
 2.1. Étape 2.1 : 2.1 Le système vérifie que les informations entrées correspondent bien aux informations fournies par un fournisseur présent dans la base de données des fournisseurs.
3. Étape 3 : Le système affiche un message de confirmation indiquant que le robot a bien été enregistré.
4. Étape 4 : Le système affiche de nouveau le menu avec les différentes options possibles.
Scénarios alternatifs :
3a. Le numéro de série est invalide (le numéro a été mal saisie).
 3a.1. Le système n’a pas trouvé de robot possédant ce numéro de série.
 3a.2. Le logiciel affiche un message d’erreur et demande au client d’entrée les données de nouveau.
 3a.3. Le scénario reprend à l’étape 2.
3b. Le type du robot entré n’est pas valide.
 3b.1. Le système n’a pas trouvé de robot de ce type.
 3b.2. Le logiciel affiche un message d’erreur et demande au client d’entrée les données de nouveau.
 3b.3. Le scénario reprend à l’étape 2.
Postconditions : Le robot est ajouté dans la liste des robots enregistrés pouvant être suivi et pouvant également être assignés à des tâches.
