Cas 4

Cas d'utilisation : Créer des tâches
But : Un client désire de créer des tâches qui seront effectués par les robots
Préconditions : Posséder un compte à Robotix
Acteurs : Client (principal)
Scénario principal :
1. Le client se connecte à Robotix.
2. Le système vérifie le compte du client.
3. Le client se rend au menu principal de l'application.
4. Le client choisit l'option de création de tâche.
5. Le client écrit les détails de la tâche en lui donnant un titre et une description.
6. Le système valide la tâche et affiche un message de confirmation en disant que la tâche a été créée.
Scénarios alternatifs :
2a. Le compte entré n'est pas enregistré sur l'application.
2a.1. Le système affiche un message d'erreur en disant qu'il n'y a pas de compte enregistré sur l'application.
2a.2. Le scénario se reprend à l'étape 1.

6a. Le système ne valide pas la tâche.
6a.1. Le système affiche un message d'erreur et demande au client de récrire à nouveau la tâche en respectant les conditions.
6a.2. Le scénario reprend à l'étape 5.
Postconditions : Le système a enregistré dans la base de données la tâche, ce qui inclut le titre et la description de cette tâche.

Cas 10

Cas d'utilisation : Participer aux activités
But : Un client souhaite de participer à diverses activités avec leur flotte de robots
Préconditions : Posséder un compte à Robotix ; Posséder au moins un robot
Acteurs : Client (principal)
Scénario principal :
1. Le client se connecte à Robotix.
2. Le système vérifie le compte du client.
3. Le client se rend au menu principal de l'application.
4. Le client choisit l'option de participer aux activités.
5. Le client choisit le type d'activité.
6. Le client choisit la catégorie de l'activité.
7. Le client choisit l'activité.
8. Le client entre le ou les noms des robots qu'il désigne pour l'activité.
9. Le système valide le choix de l'activité et le numéro du robot.
10. Le système affiche un message de confirmation en disant que le choix de l'activité a été fait.
Scénarios alternatifs :
2a. Le compte entré n'est pas enregistré sur l'application.
2a.1. Le système affiche un message d'erreur en disant qu'il n'y a pas de compte enregistré sur l'application.
2a.2. Le scénario se reprend à l'étape 1.

8a. Le client entre le mauvais nom d'un robot.
8a.1. Le système ne trouve pas de robot qui possède ce nom.
8a.2. Le système affiche un message d'erreur et demande au client de récrire à nouveau le nom du robot.
8a.3. Le scénario reprend à l'étape 8.
Postconditions : Le système a enregistré dans la base de données le choix de l'activité du client et le robot.

Cas 12

Cas d'utilisation : Trouver un fournisseur
But : Un client désire de rechercher un fournisseur dans la liste des fournisseurs enregistrés
Préconditions : Posséder un compte à Robotix ; Posséder au moins un fournisseur inscrit
Acteurs : Client (principal)
Scénario principal :
1. Le client se connecte à Robotix.
2. Le système vérifie le compte du client.
3. Le client se rend au menu principal de l'application.
4. Le client choisit l'option de rechercher un fournisseur
5. Le client choisit l'option de filtre facilitant la recherche d'un fournisseur (Ex: par nom, adresse, email, téléphone).
6. Le système valide le choix du fournisseur recherché.
Scénarios alternatifs :
2a. Le compte entré n'est pas enregistré sur l'application.
2a.1. Le système affiche un message d'erreur en disant qu'il n'y a pas de compte enregistré sur l'application.
2a.2. Le scénario reprend à l'étape 1.

6a. Le système ne trouve pas le fournisseur recherché par le client.
6a.1. Le système affiche un message d'erreur et informe au client que sa recherche ne figure pas parmi la liste des fournisseurs inscrits.
6a.2. Le scénario reprend à l'étape 5.
Postconditions : Le fournisseur fait partie de la liste des fournisseurs déjà sélectionné
